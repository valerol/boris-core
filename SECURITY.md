# Security policy

Supported release: v1.3.0 anonymized final public alpha.

Security reports should use the future repository private vulnerability reporting channel. If a repository does not yet exist, do not publish sensitive vulnerability details publicly. Open a minimal public issue requesting a private contact channel, without exploit details.

This archive intentionally contains no personal fallback email.

Security baseline: no network calls, no shell execution, no dynamic code execution, no autonomous governance, no high-stakes decision authority.
