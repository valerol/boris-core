# Real usage analysis — v1.4.4

Observed recurrent issues from this session:

1. Format drift across chat transitions and sometimes inside one chat.
2. Loss of answer counter / Mode 10 counter before EGO became default core.
3. Need to provide downloadable memory and transition artifacts after changes.
4. Risk of over-bureaucratic confirmations when operator already authorized changes.
5. Need to preserve developed physiology because it reflects real application history.

Applied hypotheses:

- Make EGO default core instead of special mode.
- Make maximum operational response format the default envelope.
- Add cross-chat restoration guard.
- Add anti-bureaucratic rule that removes redundant confirmations but preserves all contexts.
- Treat physiology as stable transit/rendering infrastructure for BOIS Core calculations.

Status: implemented in v1.4.4; requires field validation in future use.
