# BOIS overlay layers and calculation centers

Version: overlay-policy-v0.3 + core-calculation-center-policy-v0.2  
Date: 2026-06-30  
Status: general non-personal core pointer for loading additional layers.

## Purpose

BOIS:SIMA&BORIS can run without overlays or with explicitly accepted overlays. The base calculation layer is the philosophical core. Core physiology, personal adaptation layers, and project overlays organize application, memory, and procedures; substantial calculation receives a route to the philosophical center.

## Loading order

1. Load the philosophical core and core physiology according to load_order.txt.
2. Load explicitly accepted overlays.
3. Declare the order of multiple overlays.
4. Each overlay has its own human surface, manifest, evidence boundary, and rollback rule.
5. Substantial calculations return to the base philosophical center or to an explicitly declared layer philosophical center.

## Transfer boundary

A transferable project archive carries project physiology and public surface. Personal adaptation helps the operator in the internal circuit and remains in the public Self layer.

## Version history

A layer keeps current state in Current and accumulates verified boot ZIP archives in Versions. Version history remains an auditable sequence; cleanup is a separate explicit maintenance decision by the layer owner.
