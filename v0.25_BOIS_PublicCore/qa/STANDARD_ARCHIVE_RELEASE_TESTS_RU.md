# Стандартные тесты выдачи архива

Этот файл является частью физиологии Base core. Он перечисляет стандартный набор проверок, выполняемых при подготовке BOIS-архива или продукта к скачиванию/релизу.

1. Проверка идентичности архива: указаны имя архива, версия, корневая папка и package_role [роль пакета].
2. ZIP-проверка: архив открывается, CRC проходит.
3. Инвентаризация файлов: список файлов полный, намеренный и без случайных вложенных поставочных артефактов.
4. Проверка порядка загрузки: load_order, active_source_order [активный порядок источников] и границы слоя/проекта явны.
5. Проверка manifest [манифеста]: записи соответствуют содержимому архива и исключают только разрешённые сгенерированные integrity-поверхности.
6. Проверка SHA-256: каждый отслеживаемый файл имеет актуальную контрольную сумму; отказов нет.
7. Проверка JSON: все JSON-файлы разбираются, ключевые схемоподобные поля присутствуют.
8. Проверка CSV: все CSV-файлы разбираются, заголовки согласованы, строки имеют ожидаемую ширину для своей языковой поверхности.
9. Проверка владельцев правил: active rules [активные правила], STOP-сигналы, tests [тесты] и criteria [критерии] имеют владельцев и не создают скрытый дрейф владения.
10. Проверка английского канонического PDF: английский человекочитаемый физиологический PDF существует и содержит последние канонические правила слоя/проекта.
11. Проверка переводов/проекций: русские и машинные поверхности являются переводами или проекциями английского канона, а не старшим каноном.
12. PDF render [рендер PDF]: каждый PDF внутри архива рендерит все страницы без ошибок рендера.
13. DOCX render [рендер DOCX]: каждый DOCX для человека рендерится без обрезки, наложений и сломанных символов.
14. Проверка surface contract [контракта поверхности]: шапка, Detail [подробность], три терминальных копируемых блока, авторское сообщение и политики Versions/долгов присутствуют.
15. Проверка авторского сообщения: сообщение поддержки BOIS/MaOS включено, когда архив/продукт этого требует, и стоит последним в итоговом ответе.
16. Проверка local-vs-Versions [локальная проверка против внешней папки Versions]: локальная проверка архива отделена от внешней загрузки/проверки в Versions.
17. Предпроверка долга Versions: перед повтором долга загрузки/проверки целевая папка Versions проверяется, если есть доступ и известна папка.
18. Послойные долги: долги группируются по Self/Base и каждому активному проектному слою; аккаунты, площадки, сайты и канонические ссылки проекта остаются внутри проекта.
19. Философо-логическая проверка и непротиворечивость маршрутов: релиз не противоречит D/V/S, маршруту расчёта, границе полномочий или доказательности; заявленные маршруты расчёта, проверки, восстановления, публикации и долгов внутренне согласованы и совместимы друг с другом.
20. Самозамкнутость: архив содержит достаточно канонических, переводных, машинных, manifest/hash и QA-поверхностей для повторной загрузки себя.
21. Антибюрократическая проверка: новый орган/правило не остаётся, если существующий владелец чисто несёт функцию.
22. Позитивность формулировок: активные правила сначала задают желательный маршрут; ограничения защищают этот маршрут и дают ремонт, когда он возможен.
23. Смысловой корень: ремонтный релиз фиксирует корень дефекта и повторный тест смежного сценария.
24. QA-E: полевая эффективность не заявляется без внешнего полевого доказательства.
25. Финальный verify report [отчёт проверки]: создан JSON проверки со статусом PASS до выдачи архива.
26. Тест замкнутой философской машины во времени: архив симулируется как самодостаточная философская машина через загрузку, ремонт/релиз и подготовку следующего состояния. Он должен сохранять владение Base Cycle Guard, приоритет английского канона, непрерывность циклов, непротиворечивость маршрутов, путь восстановления, послойное/проектное владение и иерархию физиологий без деградации.
