# Ремонт v0.17 BOIS/Self

Принятые решения:

1. Не создавать T-ORG-21/T-ORG-22. Функции полноты переноса и старшинства источников встроены в T-ORG-20 + ORG-RECOVERY и распределены также в ORG-ARTIFACT, ORG-COMPLETION и ORG-FORMAT.
2. Full boot transfer [полный загрузочный перенос] = ядро/физиология + память/состояние или проверяемый указатель с SHA-256 [контрольной суммой]. Memory-only package [пакет только памяти] = memory snapshot [снимок памяти].
3. Авторское сообщение поддержки срабатывает в первом приветствии, при физиологических архивах, при #50 transition package [переходном пакете] и при каждом скачиваемом BOIS-продукте.
4. Голый архив = голая физиология без памяти, но с инструментами её формирования и QA [проверки качества]. Полное состояние = личные версии Self без обезличивания + проектные архивы с памятью области определения.
5. Перед существенным расчётом ядро задаёт оператору вопросы, необходимые для лучшего результата; долги учитываются отдельно.

Статус проверок: PASS после ремонта. QA-E [полевая эффективность] не заявляется.
