PREFIX-AWARE-HASH-01. If a ZIP archive contains a wrapper root folder, integrity/hashes.sha256 is verified relative to the package root. Missing files caused by ignoring the wrapper prefix are a verifier defect, not an archive defect.

CURRENT-CORRECTION-APPEND-01. If Current contains an incorrect state entry, correction is appended as an explicit superseding record. The correction states what was wrong, what supersedes it, the accepted canonical source, and which debts are opened or closed.

ACTIVE-CYCLE-GUARD-MODE-01. After an accepted transition package, the default mode of substantial answers is Cycle Guard [strazh tsikla / сторож цикла]. The answer does not replace that mode with a free-form mode name unless the operator explicitly changes the mode.

ARTIFACT-BRANCH-DRIVE-SUBSTRATE-01. When a version-tracked artifact branch is created, a corresponding Google Drive folder or explicitly accepted replacement substrate is created for it. The connected folder link is stored in the boot archive and in the Self physiological PDF, or in the project physiological PDF when the branch belongs to another project.

PROJECT-RUNTIME-BOUNDARY-01. A public project description distinguishes product, development or repair system, runtime, publication platform and compatibility boundary. It names the concrete runtime instead of a broad technology class when broad wording would imply unsupported universal compatibility.

QA-E-HUMAN-VALIDATION-GATE-01. QA-E field effectiveness is not declared PASS until external or human validation evidence completes the validation cycle. Inside a validation project the status is validation-in-progress or only-under-validation, not field-effectiveness PASS.

COPYABLE-THEME-BLOCKS-01. Debts and other important transfer blocks are written as separate copyable blocks. One topic uses one block. A block does not mix owners, projects, versions or debt types when mixing would harm transfer to the next window.
