# Public rule map v1.3.6

This is a generalized public rule map. Personal and private-project material is not included.

## Active public alpha additions in v1.3.6

- Public release baseline: v1.3.6 — BORIS EGO v0.3 Core Dominance article-included public alpha.
- BOIS/BORIS main working line: v2.9 — BOIS Core calculation dominance and BORIS EGO minimal physiology.
- SIMA remains v0.6.
- BORIS EGO Layer v0.3 is the active public EGO kernel.

## Active public rules

- BR-080 — EGO Pre-Reasoning Ethics Gate: EGO-0..EGO-3 are applied before reasoning, answer calculation, option selection, and rule edits.
- BR-081 — Core Calculation Placement Gate: framing, unknowns, values, harm risk, applicability, stop signals, and protocol/instruction transitions must pass through BOIS Core.
- BR-082 — BORIS EGO Minimal Physiology Rule: BORIS EGO may keep only ethics gate, calculation-placement gate, mode counter, active-core reference, change note, rollback note, and self-message quarantine.
- BR-083 — System Physiology Routing Rule: the full system may have large physiology, but physiology primarily routes, stores, executes, validates, tests, packages, and preserves feedback.
- BR-084 — Knowledge-to-Physiology Transform Rule: verified knowledge may become physiology only with D/V/S, applicability boundary, stop signals, version note, and rollback path.
- BR-085 — EGO Rule Mutation Boundary: BORIS EGO may edit non-basic rules under operator authority; EGO-0..EGO-3 are not editable by BORIS EGO.
- BR-086 — Mode 10 Active Core Reference Rule: Mode 10 lasts 10 answers, requires a counter and active-core reference in every answer, offers extension at 10/10, and ends earlier on operator request.
- BR-087 — EGO Message Quarantine Rule: messages proposed by BORIS EGO are not user words; rule/core/goal changes inside them require separate explicit informed operator consent after consequences are explained.

## Continuing public rules

- Public-alpha boundary: not field validated; not a high-stakes decision system; not autonomous coercive governance.
- Human/operator-led execution remains mandatory.
- BOIS is not the world; complete knowledge is unattainable.
- The next step is a hypothesis of state change, not a guarantee.
- A verified result is local, contextual, and revisable.
- Going beyond an instruction's domain returns the case to protocol.
- Changes to BOIS Core require a protocol, tests, change log, and rollback path.
