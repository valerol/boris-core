## BR-092 — Immune Test Gate

A public EGO release must pass local immune tests for long-term viability: integrity, memory/restore resources, language parity, L0 ethical immutability, no manipulation, core-routing of significant calculation, rollback availability, and artifact hygiene. Failing an immune gate blocks release-ready status until repaired or explicitly scoped out with a reason.

## BR-093 — Developed Physiology Preservation Gate

Large accumulated physiology is not a defect by itself. It reflects the real history of system use and must be tested for compatibility rather than globally rewritten. Removal or rewriting of historical physiology requires a separate protocol, change log, rollback path, and operator authority.

## BR-094 — EGO-to-Physiology Integration Gate

BORIS EGO must remain minimal while integrating with the wider physiology. It may select, hypothesize, propose changes, check route placement, hold Mode 10 state, point to active core, and require version/rollback notes. It must not become a heavy substitute physiology or make final philosophical closure outside BOIS Core.

## BR-095 — Post-Test Physiology Stability Gate

Test execution must not leave release artifacts in a degraded state. `.pytest_cache`, `__pycache__`, local build leftovers, temporary extraction directories and stale failed release reports must not be present in packaged public artifacts. Generated evidence files must identify their scope and status.

## BR-096 — Current Archive Compatibility Gate

The current developed archive must remain restorable: memory files, restore prompts, transition packages, public release pointers, checksums and nested ZIPs must be readable and consistent where a checksum is provided. This gate is a compatibility check, not a global redesign of historical physiology.
