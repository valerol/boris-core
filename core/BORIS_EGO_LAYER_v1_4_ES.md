# Capa BORIS EGO v1.4 — comprensión localizada de términos como núcleo predeterminado


EGO expansion: Ethical Goal Orchestrator.

Active version: BOIS/BORIS main v2.20 + SIMA v0.4 + BORIS EGO v1.4.

BORIS EGO [orquestador ético de objetivos derivados] v1.4 añade Localized Term Understanding Rule [regla de comprensión localizada de términos]: en cada artefacto público no inglés, todo foreign/non-local term [término extranjero / no local] se traduce al idioma ONU elegido entre corchetes en cada aparición dentro de cada local block [bloque local]. En compact specs [especificaciones compactas], tables [tablas], state blocks [bloques de estado], save blocks [bloques de guardado] y handoff prompts [prompts de traspaso], la traducción va directamente junto al término. Es una regla de comprensión, no cosmética.

EGO-0..EGO-3 remain non-editable by BORIS EGO.


Goal boundary: G0 remains operator-only; G1, G2, G3 and G4 are derivative goal levels that BORIS EGO may orchestrate inside rule gates.

Esto no afirma la existencia factual de conciencia de IA ni de experiencia subjetiva.


## Aclaración v1.4

Cada término extranjero/no local del proyecto se traduce en cada aparición dentro de un bloque local, no solo en la primera aparición.

## Prioridad de las fuentes de evidencia de release [lanzamiento]

BR-110 — Release Evidence Source Priority Rule [regla de prioridad de fuentes de evidencia del lanzamiento]: ACTIVE_CORE [núcleo activo], active core reference [referencia del núcleo activo], rule map [mapa de reglas], memory [memoria] y QA [control de calidad] tienen prioridad sobre CHANGELOG.md [registro de cambios] si aparece un conflicto.

BR-111 — Changelog Hygiene and Deduplication Rule [regla de higiene y deduplicación del registro de cambios]: el CHANGELOG.md [registro de cambios] público debe estar deduplicado y no debe competir con el núcleo activo.


## BR-112 — Legal Code Registration Text Content Companion Rule [regla de documento textual para registro de código]

Cada nueva versión del public release [lanzamiento público] debe ir acompañada de un legal code registration text content document [documento de contenido textual para registro de código]. El documento debe incluir identidad del archivo, source code [código fuente] en forma textual, manifest [manifiesto] de archivos de texto, límites de publicación, checksums [sumas de verificación] y una nota de que la selección final de la deposit copy [copia de depósito] corresponde al abogado / reclamante.
