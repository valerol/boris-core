# BOIS calculation-center and layer relation policy

Version: core-calculation-center-policy-v0.2  
Date: 2026-06-30  
Status: general core policy; personal and project facts live in their own layers.

## Purpose

The base calculation layer of BOIS:SIMA&BORIS is the philosophical core. Core physiology, personal adaptation layers, and project overlays organize application, memory, organs, stops, interfaces, and artifacts. Substantial calculation receives a route to the base philosophical center or to an explicitly declared layer philosophical center.

## Core rules

- CC-R01. Substantial calculation starts in or returns to the philosophical core: meaning, D/V/S, evidence, unknowns, risk, agency, closure, and responsibility are checked before execution.
- CC-R02. Rules, PDFs, memory, tables, and project procedures route conclusions to the base philosophical center or to an explicitly declared layer center.
- CC-R03. Project layers are overlays with their own goals, memory, accounts, strategies, channels, game procedures, or domain procedures.
- CC-R04. A layer philosophical machine becomes active after declaring domain, D/V/S, inputs, outputs, owner, evidence boundary, stop signals, and relation to the base philosophical center.
- CC-R05. Project calculations return to the common circuit: base core, project center in its domain, or operator with marked unknowns and responsibility boundary.
- CC-R06. Personal adaptation layers help an operator use the core and keep a separate folder, physiology, memory, manifest, and transfer boundary.
- CC-R07. Transferable project archives carry project physiology and public surface; personal calibration stays in the operator's private circuit.
- CC-R08. Current stores current layer state; Versions accumulates verified boot ZIP archives as auditable history.

QA-E field effectiveness remains a future metric until field data exists.
